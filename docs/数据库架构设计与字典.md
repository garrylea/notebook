# 智能便签任务管理系统 —— 数据库架构设计与数据字典

> **文档说明**
> 本文档基于《需求规格说明书_v1.2》和《接口设计文档》输出，定位于“阶段二：系统详细设计”，为后端研发提供底层 MySQL / PostgreSQL 关系型数据库的建表依据。

---

## 1. 实体关系概述 (E-R 简述)

本系统的核心领域模型围绕“用户”与“便签”展开。主要实体及关系如下：

1.  **用户 (User)** `1:N` 拥有 **便签 (Note)**
2.  **便签 (Note)** `1:N` 拥有 **子任务 (Subtask)** （子任务支持自关联，形成树形结构，最大深度3层）
3.  **便签 (Note)** `1:N` 拥有 **附件 (Attachment)**
4.  **便签 (Note)** `1:N` 拥有 **操作日志 (Activity_Log)** （记录状态流转与变更历史）
5.  *(预留)* **便签 (Note)** `M:N` 共享给 **用户 (User)** （V1.2 协作特性）

### 1.1 全局设计规约
*   **主键策略：** 统一采用分布式发号器（如雪花算法 Snowflake）或 UUID，考虑到 MySQL InnoDB 索引性能，推荐 `BIGINT` 配合雪花算法，或 `CHAR(36)` 的 UUID（视开发语言便利度，本设计妥协为更通用的 `VARCHAR(32)` 或 `CHAR(36)`）。
*   **字符集：** 强制 `utf8mb4`，排序规则 `utf8mb4_unicode_ci`，以支持 Emoji 写入。
*   **软删除：** 绝大多数核心表使用 `is_deleted` (TINYINT) 作为逻辑删除标记，防止历史数据审计断链。
*   **通用审计字段：** 每张表必备 `created_at` (创建时间), `updated_at` (更新时间)。

## 2. 数据字典 (Data Dictionary)

### 2.1 用户表 (`sys_user`)
存储用户的基本认证与配置信息。

| 字段名          | 数据类型     | 约束/索引    | 默认值                      | 描述                            |
| --------------- | ------------ | ------------ | --------------------------- | ------------------------------- |
| `id`            | VARCHAR(36)  | PK           |                             | 用户全局唯一ID                  |
| `username`      | VARCHAR(50)  | UK, NOT NULL |                             | 登录用户名                      |
| `email`         | VARCHAR(100) | UK           | NULL                        | 邮箱地址                        |
| `password_hash` | VARCHAR(255) | NOT NULL     |                             | BCrypt/Argon2 密码哈希值        |
| `avatar_url`    | VARCHAR(255) |              | NULL                        | 用户头像路径                    |
| `preferences`   | JSON         |              | NULL                        | 用户偏好设置 (如语言、主题)     |
| `status`        | TINYINT      | NOT NULL     | 1                           | 账号状态: 1-正常, 0-禁用        |
| `created_at`    | DATETIME     | NOT NULL     | CURRENT_TIMESTAMP           | 注册时间                        |
| `updated_at`    | DATETIME     | NOT NULL     | ON UPDATE CURRENT_TIMESTAMP | 更新时间                        |
| `is_deleted`    | TINYINT(1)   | NOT NULL     | 0                           | 软删除标识 (0-未删除, 1-已删除) |

### 2.2 便签主表 (`busi_note`)
系统的核心业务表。

| 字段名       | 数据类型     | 约束/索引     | 默认值                      | 描述                                                           |
| ------------ | ------------ | ------------- | --------------------------- | -------------------------------------------------------------- |
| `id`         | VARCHAR(36)  | PK            |                             | 便签全局唯一ID                                                 |
| `user_id`    | VARCHAR(36)  | IDX, NOT NULL |                             | 所属用户ID                                                     |
| `title`      | VARCHAR(100) | NOT NULL      |                             | 便签标题                                                       |
| `content`    | TEXT         |               | NULL                        | 便签正文 (Markdown 纯文本)                                     |
| `color`      | VARCHAR(20)  | NOT NULL      | 'blue'                      | 颜色语义值 (red, orange, blue, green)                          |
| `status`     | VARCHAR(20)  | IDX, NOT NULL | 'todo'                      | 状态枚举 (todo, doing, done, hanging, archived)                |
| `due_date`   | DATETIME     | IDX           | NULL                        | 截止日期（**选填**，为 NULL 时不在日历中显示，仅在主列表可见） |
| `pin_top`    | TINYINT(1)   | NOT NULL      | 0                           | 是否置顶 (0-否, 1-是)                                          |
| `created_at` | DATETIME     | IDX, NOT NULL | CURRENT_TIMESTAMP           | 创建时间 (常用于瀑布流排序)                                    |
| `updated_at` | DATETIME     | NOT NULL      | ON UPDATE CURRENT_TIMESTAMP | 更新时间                                                       |
| `is_deleted` | TINYINT(1)   | NOT NULL      | 0                           | 软删除标识                                                     |

### 2.3 子任务表 (`busi_subtask`)
支持树形层级结构的子任务存储。

| 字段名         | 数据类型     | 约束/索引     | 默认值                      | 描述                                     |
| -------------- | ------------ | ------------- | --------------------------- | ---------------------------------------- |
| `id`           | VARCHAR(36)  | PK            |                             | 子任务ID                                 |
| `note_id`      | VARCHAR(36)  | IDX, NOT NULL |                             | 关联的主便签ID                           |
| `parent_id`    | VARCHAR(36)  | IDX           | NULL                        | 父级子任务ID (若为最顶层子任务则为 NULL) |
| `title`        | VARCHAR(200) | NOT NULL      |                             | 子任务内容描述                           |
| `is_completed` | TINYINT(1)   | NOT NULL      | 0                           | 完成状态 (0-未完成, 1-已完成)            |
| `sort_order`   | INT          | NOT NULL      | 0                           | 排序值 (用于同级拖拽排序)                |
| `created_at`   | DATETIME     | NOT NULL      | CURRENT_TIMESTAMP           |                                          |
| `updated_at`   | DATETIME     | NOT NULL      | ON UPDATE CURRENT_TIMESTAMP |                                          |

> **业务实现注:** 代码应用层负责限制 `parent_id` 构成的树最大深度不超过 3 层，避免无限递归。

### 2.4 附件元数据表 (`busi_attachment`)
记录上传的文件块合并后的物理入口及元信息。

| 字段名       | 数据类型     | 约束/索引     | 默认值            | 描述                                |
| ------------ | ------------ | ------------- | ----------------- | ----------------------------------- |
| `id`         | VARCHAR(36)  | PK            |                   | 附件记录ID                          |
| `note_id`    | VARCHAR(36)  | IDX, NOT NULL |                   | 挂载的便签ID                        |
| `file_name`  | VARCHAR(255) | NOT NULL      |                   | 用户上传的原始文件名                |
| `file_size`  | BIGINT       | NOT NULL      |                   | 文件大小 (字节)                     |
| `file_type`  | VARCHAR(50)  | NOT NULL      |                   | MIME 类型 (如 image/png, video/mp4) |
| `file_path`  | VARCHAR(500) | NOT NULL      |                   | 服务器本地存储相对路径或 OSS URL    |
| `md5_hash`   | VARCHAR(32)  | IDX           | NULL              | 文件 MD5 (用于断点续传/秒传校验)    |
| `created_at` | DATETIME     | NOT NULL      | CURRENT_TIMESTAMP | 上传时间                            |
| `is_deleted` | TINYINT(1)   | NOT NULL      | 0                 | 逻辑删除标识                        |

### 2.5 活动变更日志表 (`sys_activity_log`)
用作 Timeline 数据源，记录关键操作溯源。属于冷读热写表，数据量可能极大，需考虑定期归档策略。

| 字段名        | 数据类型    | 约束/索引      | 默认值            | 描述                                             |
| ------------- | ----------- | -------------- | ----------------- | ------------------------------------------------ |
| `id`          | BIGINT      | PK (自增/雪花) |                   | 流水自增ID                                       |
| `note_id`     | VARCHAR(36) | IDX, NOT NULL  |                   | 发生变更的便签ID                                 |
| `operator_id` | VARCHAR(36) | NOT NULL       |                   | 操作人用户ID                                     |
| `action_type` | VARCHAR(50) | NOT NULL       |                   | 操作动作 (如 UPDATE_STATUS, UPLOAD_FILE)         |
| `detail`      | JSON        |                | NULL              | 变更明细详情 (如 {"from": "todo", "to": "done"}) |
| `created_at`  | DATETIME    | IDX, NOT NULL  | CURRENT_TIMESTAMP | 发生时间                                         |

*(注：协作相关的数据表设计，如 `busi_note_share` 权限表，将在 V1.2 迭代时补齐，MVP 阶段暂不深入。)*

## 3. SQL 核心查询优化预估

为了支撑核心界面的快速渲染，需在建表时确保以下复合索引的最佳实践：
1.  **便签列表/瀑布流加载**：需根据 `user_id`, `is_deleted=0` 过滤，并依据 `created_at` 倒序分页。建议建立联合索引 `idx_user_created (user_id, is_deleted, created_at)`。
2.  **日历视图快查**：需提取某月内 `due_date` 不为空的数据。建议建联合索引 `idx_user_due (user_id, is_deleted, due_date)`。
