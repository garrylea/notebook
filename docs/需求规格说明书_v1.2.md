## 1. 项目概述

### 1.1 背景

用户需要一款**多用户、跨平台、自部署**的任务便签管理工具，以便签（卡片）形式组织日常任务，支持协作、全生命周期追踪、附件管理、日历与统计功能。

### 1.2 目标

| 目标           | 说明                                    |
| -------------- | --------------------------------------- |
| 高效记录       | 用户能在 3 秒内创建一张新便签           |
| 可视化管理     | 通过颜色语义、状态、日历直观呈现任务    |
| 全生命周期追踪 | 记录便签从创建到归档的每一次变更        |
| 跨平台一致性   | 在 Windows、macOS、主流浏览器上体验一致 |
| 协作支持       | 多用户可共享便签并进行协同编辑          |

### 1.3 范围

**Web 应用（前后端分离，RESTful API）**，服务端自部署，附件存储于服务器本地文件系统。暂不实现账务管理，后续版本按需追加。

## 2. 术语定义

| 术语                          | 定义                                  |
| ----------------------------- | ------------------------------------- |
| **便签 (Note)**               | 基本记录单元，对应一项任务            |
| **子任务 (Subtask)**          | 归属于某便签的子级任务，最多 3 层嵌套 |
| **附件 (Attachment)**         | 挂载在便签上的文件资源                |
| **标签 (Tag)**                | 用于分类便签的关键词                  |
| **重复任务 (Recurring Note)** | 按设定频率自动创建新实例的循环任务    |
| **状态 (Status)**             | 便签当前所处的生命周期阶段            |

## 4. 功能需求

### 4.1 便签管理

#### 4.1.1 创建便签

- 通过悬浮按钮（FAB）或快捷键（`Ctrl/Cmd + N`）一键创建。
- 创建时**必填**：标题、颜色；**选填**：截止日期（用户尚不确定完成时间时可留空）、正文、标签、优先级、子任务。
- 系统自动记录 **创建时间（created_at）**，初始状态为 **"进行中（In Progress）"**（直接进入活跃工作区，无需额外状态切换）。
- 无截止日期的便签在日历视图中**不显示**，仅在主列表工作区中展示。

#### 4.1.2 编辑便签

- 正文使用 **Markdown 格式**，提供编辑/预览双模式（或左右分栏）。
- 提供 Markdown 快捷工具栏（一键插入标题、加粗、列表、代码块等）。
- 每次保存自动更新 **修改时间（updated_at）**，并保留完整修改历史。

#### 4.1.3 颜色标记（固定语义）

系统提供 **8 种固定颜色**，颜色含义预定义，不支持自定义：

| 颜色   | 语义            |
| ------ | --------------- |
| 🔴 红色 | 紧急            |
| 🟠 橙色 | 重要但不紧急    |
| 🟡 黄色 | 待跟进 / 需注意 |
| 🟢 绿色 | 顺利进行中      |
| 🔵 蓝色 | 常规任务        |
| 🟣 紫色 | 个人事务        |
| ⚫ 灰色 | 低优先级        |
| ⚪ 白色 | 默认（无分类）  |

颜色说明在选色器旁以 tooltip 展示，所有视图中颜色表现一致。

#### 4.1.4 完成与删除

- 标记"**已完成**"后，便签从主工作区移出，进入**"历史记录"分区**，标题显示 ~~删除线~~。
- 软删除（移至**垃圾桶**分区），记录 **删除时间（deleted_at）**；回收站数据保留 30 天，支持恢复或永久删除（二次确认）。

#### 4.1.5 搜索与筛选

- 全文搜索（标题 + 正文），搜索范围覆盖所有分区。
- 按状态、颜色、标签、优先级、日期范围多条件筛选。
- 主工作区默认排序：**优先级（高→低）→ 创建时间（新→旧）**；支持切换为截止时间、修改时间排序。

#### 4.1.6 主视图状态分区展示（核心交互逻辑）

系统采用**单页应用（SPA）**架构，点击左侧导航各分区图标时，主内容区**原地切换**内容，无页面跳转。

| 分区                 | 收录状态                              | 左侧入口                     | 排序规则                  |
| -------------------- | ------------------------------------- | ---------------------------- | ------------------------- |
| **主工作区**（默认） | 进行中 (In Progress)、草稿 (Draft)    | 主导航图标（高亮）           | 优先级降序 → 创建时间降序 |
| **历史记录**         | 已完成 (Completed)、已归档 (Archived) | 次级导航图标                 | 完成时间降序              |
| **悬挂列表**         | 挂起 (Suspended)                      | 次级导航图标（角标显示数量） | 挂起时间降序              |
| **垃圾桶**           | 已删除 (Deleted)                      | 底部图标                     | 删除时间降序              |

**设计原则：**
- 点击左侧任意分区图标 → 主内容区**原地**切换对应便签列表，左侧当前入口高亮。
- 悬挂列表通过**角标数量**提醒用户存在待恢复的挂起任务。
- 截止日期不再作为主线，便签以**优先级 + 创建时间**作为主排序依据。

#### 4.1.7 主工作区渐进式折叠加载规则

为避免大量便签同屏带来视觉压迫，主工作区（以及各分区）采用**渐进式折叠加载**策略：

- **默认展示**：进入任意分区时，仅展示前 **10 条**（按当前分区排序规则）。
- **折叠收起**：第 10 条之后的便签默认**不渲染**，列表底部显示"**展开更多（还有 N 条）**"按钮。
- **按需加载（Load More）**：
  - 用户点击"**展开更多**"按钮 → 加载并追加展示接下来的 **10 条**。
  - 用户向下**滚动至页面底部** → 自动触发加载下一批 10 条（无限滚动），无需点击按钮。
- **全量加载完毕**：所有便签加载完时，底部显示"**已加载全部 N 条**"提示，按钮消失。
- **注意**：本策略为前端视图层的渐进展示，后端 API 仍通过标准分页接口（`page` + `limit=10`）按需拉取数据。

### 4.2 任务与子任务

#### 4.2.1 子任务结构

- 每张便签可添加多个子任务，支持 **最多 3 层嵌套**。
- 子任务字段：描述文本（Markdown）、状态、负责人（可选）、截止日期（可选）。
- 便签卡片上以**进度条**展示子任务完成情况（已完成数 / 总数）。

#### 4.2.2 子任务状态

| 状态   | 说明                 |
| ------ | -------------------- |
| 待办   | 尚未开始             |
| 进行中 | 处理中               |
| 已完成 | 完成                 |
| 已取消 | 取消，不计入进度分母 |

#### 4.2.3 拖拽排序

便签详情页内，子任务支持拖拽调整顺序。

### 4.3 便签状态管理

| 状态                     | 说明                   | 触发条件                         |
| ------------------------ | ---------------------- | -------------------------------- |
| **草稿 (Draft)**         | 内容未完善             | 创建时默认                       |
| **进行中 (In Progress)** | 任务已开始             | 用户手动切换                     |
| **挂起 (Suspended)**     | 暂时搁置，等待外部条件 | 用户手动切换，可填写挂起原因     |
| **已完成 (Completed)**   | 任务完成               | 用户标记，标题显示删除线         |
| **已归档 (Archived)**    | 不再活跃，仅供查阅     | 完成 30 天后自动归档，或手动归档 |
| **已删除 (Deleted)**     | 移入回收站             | 用户删除                         |

- 每次状态变更记录：操作时间、操作人、变更前/后状态、备注。
- 状态历史在便签详情页"时间线"标签中展示。

### 4.4 附件管理

#### 4.4.1 存储方式

- 附件存储于**服务器本地文件系统**，路径结构：`/uploads/{user_id}/{note_id}/{filename}`。
- 服务端统一管理访问权限，禁止未授权直接访问文件路径。

#### 4.4.2 支持的附件类型

| 类型      | 格式                            | 功能                     |
| --------- | ------------------------------- | ------------------------ |
| 纯文本    | `.txt`, `.md`, `.csv`           | 在线预览全文             |
| Word 文档 | `.docx`, `.doc`                 | 仅下载，不支持在线编辑   |
| 音频      | `.mp3`, `.wav`, `.ogg`, `.m4a`  | 内嵌播放器在线播放       |
| 视频      | `.mp4`, `.webm`, `.mov`         | 内嵌播放器在线播放       |
| 图片      | `.jpg`, `.png`, `.gif`, `.webp` | 缩略图预览，点击放大     |
| 其他      | 任意格式                        | 仅下载，显示文件类型图标 |

#### 4.4.3 附件操作

- **上传**：拖拽或点击选择，支持批量，实时显示进度。
- **预览/播放**：文本、图片、音视频在页面内预览，无需下载。
- **下载**：所有附件支持一键下载。
- **删除**：二次确认后，同步从文件系统物理删除。

#### 4.4.4 存储限制

- 单个附件最大 **100 MB**；每用户总存储上限 **10 GB**（管理员可配置）。

### 4.5 日历视图与提醒

- **便签以截止日期（due_date）在日历中定位**；截止日期为**选填**，无截止日期的便签不在日历中显示，仅在主列表工作区可见。用户可在便签详情中随时补填截止日期，填写后即出现在日历对应日期格。
- 提供**日 / 周 / 月**视图切换，前后翻页，"回到今天"按钮。
- 便签以对应颜色块展示，点击快速预览详情。
- 支持在日历中拖动便签以更改截止日期。

#### 4.5.1 定时提醒

- 提前提醒时间可选：**15 分钟 / 1 小时 / 1 天 / 自定义**。
- 提醒方式：**Web Push 系统通知** + 应用内消息中心；后续可扩展邮件。
- 任务逾期未完成时，每天重复推送一次，直至完成或用户关闭该提醒。

---

### 4.6 重复任务

- 便签可设置为**重复任务**，按频率自动生成新实例。
- 重复频率：**每天 / 每周（指定星期几）/ 每月（指定日期）/ 自定义间隔**。
- 结束条件：指定结束日期 或 永不结束。
- 完成/删除某一实例**不影响**后续实例。
- 日历中各实例独立显示，带循环标记 🔁。

---

### 4.7 统计与分析

> 账务相关统计项暂不实现。

统计模块分两个层次：**全局仪表盘**（左侧"统计图表"入口）与**历史便签专项分析**（"历史记录"分区内嵌看板）。

#### 4.7.1 全局仪表盘统计维度

| 维度             | 图表类型          | 说明                         |
| ---------------- | ----------------- | ---------------------------- |
| 当前状态分布     | 饼图              | 进行中/挂起/完成/归档占比    |
| 新建 vs 完成趋势 | 折线图（按日/周） | 生产效率对比                 |
| 颜色分布         | 条形图            | 各优先级颜色的任务量         |
| 标签分布         | 标签云 + 条形图   | 高频标签识别                 |
| 优先级分布       | 饼图              | 紧急/重要/常规/低优占比      |
| 逾期分析         | 列表              | 含截止日期且超时未完成的便签 |

时间范围：**周 / 月 / 年** 切换，支持自定义区间。

#### 4.7.2 历史便签专项统计（History Analytics）

用户进入"历史记录"分区后，顶部提供**可折叠的统计分析看板**，从多角度回顾已完成任务：

| 统计维度        | 图表类型                        | 说明                                     |
| --------------- | ------------------------------- | ---------------------------------------- |
| 完成数量趋势    | 折线图/柱状图（日/周/月切换）   | 每个周期完成了多少便签                   |
| 平均处理时长    | 横向条形图（按颜色/优先级分组） | 从创建到完成的平均天数                   |
| 按颜色/类型分类 | 饼图                            | 哪种类型任务完成得最多                   |
| 按标签分类排行  | 横向条形图                      | 各标签下完成便签数排行                   |
| 月度完成热力图  | GitHub Contribution 风格热力图  | 哪些天任务完成最集中                     |
| 逾期完成分析    | 数字卡片 + 列表                 | 已完成中超出截止日期的比例及平均逾期天数 |

**交互说明：**
- 历史分析看板默认**折叠**，顶部有"📊 展开统计分析"按钮。
- 时间筛选：**本周 / 本月 / 本年 / 自定义**，与下方历史便签列表联动同步过滤。

#### 4.7.3 报表导出

- 导出格式：**PDF** 或 **CSV**。
- 支持按自定义日期范围生成，可选导出全局统计或仅历史记录统计。

### 4.8 协作功能

- 便签所有者可邀请其他注册用户协作，权限分两级：**只读 / 可编辑**。
- 可编辑协作者可修改正文、子任务状态；**不可**删除便签、修改颜色或截止日期（仅所有者）。
- 邀请方式：通过用户名或邮箱搜索，被邀请者收应用内通知。
- 所有者可随时撤销协作权限。

---

### 4.9 多客户端支持与数据同步

#### 4.9.1 支持平台

| 平台         | 支持方式                                       |
| ------------ | ---------------------------------------------- |
| macOS        | Web 浏览器（Chrome / Safari / Firefox / Edge） |
| Windows      | Web 浏览器（Chrome / Edge / Firefox）          |
| 主流浏览器   | Web 应用（PWA，可安装到桌面）                  |
| 移动端（v2） | 响应式 Web                                     |

#### 4.9.2 数据同步

- 服务端**自部署**，所有数据存于服务器，多端通过 WebSocket 或 SSE 实时同步。
- **离线模式**：缓存最近数据，网络恢复后自动同步，冲突以服务端为准并提示用户。
- 支持数据全量导出（JSON 格式）。

## 5. 非功能需求

### 5.1 性能

| 指标                      | 目标                           |
| ------------------------- | ------------------------------ |
| 页面首次加载（FCP）       | ≤ 2 秒（3G 网络）              |
| 便签列表加载（≤ 1000 条） | ≤ 1 秒                         |
| 搜索响应时间              | ≤ 500 ms                       |
| 附件上传进度反馈          | 上传开始后 300 ms 内显示进度条 |

### 5.2 安全

- 用户数据严格隔离，用户只能访问自己及被共享的数据。
- 全站 HTTPS（TLS 1.3）。
- 账号仅支持**邮箱 + 密码**注册与登录（v1 不接入第三方 OAuth）。
- 附件访问需经服务端鉴权，禁止直接路径访问。
- 敏感操作（登录、导出、删除）记录操作日志。

### 5.3 可靠性

- 可用性目标：≥ 99.5%（月度）。
- 数据每日全量备份，保留 30 天。
- 回收站数据保留 30 天后自动清除。

### 5.4 可维护性

- 前后端分离，API 遵循 RESTful 规范，提供 OpenAPI 文档。
- 核心业务逻辑代码覆盖率目标 ≥ 80%。

## 6. 数据模型

### 6.1 便签（Note）

```
Note {
  id              UUID (PK)
  user_id         UUID (FK -> User)
  title           String (max 200)
  content         Text (Markdown)
  color           Enum [red, orange, yellow, green, blue, purple, gray, white]
  priority        Enum [low, medium, high, urgent]
  status          Enum [draft, in_progress, suspended, completed, archived, deleted]
  tags            String[]
  due_date        DateTime?       -- 选填；无截止日期则不在日历中显示
  reminder_offset Integer?        -- 提前提醒分钟数
  is_recurring    Boolean
  created_at      DateTime
  updated_at      DateTime
  completed_at    DateTime?
  deleted_at      DateTime?
  archived_at     DateTime?
}
```

### 6.2 子任务（Subtask）

```
Subtask {
  id              UUID (PK)
  note_id         UUID (FK -> Note)
  parent_id       UUID?           -- 支持最多 3 层嵌套
  title           String (max 500, Markdown)
  status          Enum [todo, in_progress, completed, cancelled]
  assignee        String?
  due_date        DateTime?
  sort_order      Integer
  created_at      DateTime
  updated_at      DateTime
}
```

### 6.3 附件（Attachment）

```
Attachment {
  id              UUID (PK)
  note_id         UUID (FK -> Note)
  filename        String
  file_type       String (MIME type)
  file_size       Integer (bytes)
  storage_path    String          -- 服务器本地文件路径
  uploaded_at     DateTime
  uploaded_by     UUID (FK -> User)
}
```

### 6.4 状态变更日志（StatusLog）

```
StatusLog {
  id              UUID (PK)
  note_id         UUID (FK -> Note)
  from_status     Enum
  to_status       Enum
  changed_by      UUID (FK -> User)
  changed_at      DateTime
  remark          String?
}
```

### 6.5 重复规则（RecurrenceRule）

```
RecurrenceRule {
  id              UUID (PK)
  note_id         UUID (FK -> Note)
  frequency       Enum [daily, weekly, monthly, custom]
  interval        Integer         -- 每隔 N 个周期
  week_days       Integer[]?      -- weekly: 0=周日, 1=周一, ...
  month_day       Integer?        -- monthly: 指定日期 1-31
  ends_at         DateTime?       -- null = 永不结束
  created_at      DateTime
}
```

### 6.6 协作关系（NoteCollaborator）

```
NoteCollaborator {
  id              UUID (PK)
  note_id         UUID (FK -> Note)
  user_id         UUID (FK -> User)
  permission      Enum [read, edit]
  invited_at      DateTime
  accepted_at     DateTime?
}
```

## 7. 便签状态流转

```
              创建
             ────▶  [草稿 Draft]
                         │
                         │ 开始处理
```

## 8. UI/UX 设计原则

1. **直观优先**：便签卡片为核心视觉元素，颜色语义、状态、子任务进度一目了然。
2. **快速操作**：创建、完成、删除等高频操作不超过 2 次点击。
3. **响应式布局**：适配桌面（≥ 1200px）、平板（768–1199px）、手机（< 768px）。
4. **深色 / 浅色模式**：跟随系统设置，用户可手动覆盖。
5. **无障碍**：符合 WCAG 2.1 AA，支持键盘导航。
6. **本地化**：支持**简体中文**和**英文**，界面语言、提醒通知语言均跟随用户语言偏好设置自动切换，日期/数字格式亦随之适配。
