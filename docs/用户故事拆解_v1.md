# 智能便签任务管理系统 —— 用户故事（User Story）拆解说明

> **文档说明**
> 此文档基于《需求规格说明书_v1.2》编制，将纯文本的功能需求转化为敏捷开发中的 Epic（史诗级需求）和 Story（用户故事）。
> 格式规范：作为 【角色】，我希望 【功能】，以便于 【价值/目的】。
> 验收标准（AC）：定义开发完成后，测试通过的必要边界条件。

---

## Epic 1: 便签主体与状态管理
**描述：** 用户通过创建带颜色的卡片（便签）来记录任务，并控制其生命周期的状态流转，这是系统的核心脉络。

### Story 1.1: 快速且全要素地创建便签
作为 **普通用户**，我希望能够 **点击新建或快捷键新建一张便签（标题、颜色必填，截止时间选填）**，以便于 **我能快速记录一笔任务，且在3秒内无阻滞地完成创建，尚不清楚完成时间时不被强制填写。**
- **验收标准 (AC):**
  - [ ] 点击"新建"按钮，弹出新建抽屉或弹框。
  - [ ] 必填项：标题（≤50字）、颜色（默认色兜底）；截止日期为**选填**，允许留空保存。如果标题为空，点击保存时拦截并红字提示。
  - [ ] 正文支持 Markdown（包括粗体、斜体、列表、代码块），保存后可以渲染。
  - [ ] 创建成功后，该便签生成唯一 ID，初始状态为"**进行中**"，出现在**主工作区**列表顶部（按优先级+时间排序）。
  - [ ] 无截止日期的便签成功创建后，在日历视图中**不显示**，仅在主工作区列表可见。

### Story 1.2: 便签颜色与视觉分类
作为 **普通用户**，我希望能够 **为每张便签指定不同颜色（紧急红、重要橙、常规蓝、低优绿）**，以便于 **我能在列表/日历中根据颜色快速识别任务优先级或类型。**
- **验收标准 (AC):**
  - [ ] 新建和编辑时，可从系统提供的颜色画板选色（固定色值）。
  - [ ] 列表视图中，便签卡片的背景色或边界醒目展示该颜色语义。

### Story 1.3: 任务状态标记与分区流转
作为 **普通用户**，我希望能够 **随时更改便签状态（完成/挂起/删除），完成后便签自动移至历史记录，挂起后移至悬挂列表，删除后移至垃圾桶**，以便于 **主工作区始终清爽，只关注当前需要处理的事项。**
- **验收标准 (AC):**
  - [ ] 在便签卡片底部，有"✅ 完成"快捷按钮，点击后便签**立即从主工作区消失**，进入"历史记录"分区，标题显示删除线。
  - [ ] 在便签卡片底部，有"⏸ 挂起"快捷按钮，点击后便签进入"悬挂列表"分区；左侧导航"悬挂列表"入口角标数量 +1。
  - [ ] 在详情页内，可以通过状态下拉菜单手动切换各状态。
  - [ ] 在"历史记录"分区，用户可将已完成便签重新激活（恢复为"进行中"），便签返回主工作区。

### Story 1.4: 操作历史溯源
作为 **系统管理员/普通用户**，我希望 **便签能自动记录并展示所有的变更生命周期**，以便于 **团队协作时或个人复盘时，知晓“谁在何时修改了什么”。**
- **验收标准 (AC):**
  - [ ] 每次生成、修改标题正文、修改状态、修改颜色、增加附件等操作，系统自动记录。
  - [ ] 在便签详情页的侧边栏或底部标签中，提供“活动时间线（Timeline）”视图，按时间倒序排列展示记录。

## Epic 2: 子任务体系与递归展示
**描述：** 对于大型任务便签，用户需要将其拆分为多个层层嵌套的小任务。

### Story 2.1: 单个便签下的子任务拆分
作为 **普通用户**，我希望能够 **为一张大型便签添加多条子任务，并支持它们各自的勾选完成**，以便于 **将庞大的工作目标化为可执行的细微步骤。**
- **验收标准 (AC):**
  - [ ] 便签详情内支持“添加子任务”按钮，录入文本即生成带 checkbox 的条目。
  - [ ] 子任务勾选后即标为完成，主便签外部应展示“完成度比例（如 3/5 进度条）”。
  - [ ] 删除子任务有二次确认。

### Story 2.2: 子任务的嵌套（3 层级）
作为 **高级用户**，我希望能够 **将某个子任务再细分为下一级子任务（最多支持 3 层）**，以便于 **应对极度复杂的项目规划结构。**
- **验收标准 (AC):**
  - [ ] 第一级子任务条目下，可继续缩进添加第二级、第三级子任务。
  - [ ] 下级全部完成后，上一级自动计算完成度或联动状态。
  - [ ] 超过 3 级嵌套时，UI 提示“已达到最大嵌套层级”，禁用继续新建子层的按钮。

## Epic 3: 附件长传与预览
**描述：** 任务执行中需要补充背景资料或证明材质，需上传多媒体附件。

### Story 3.1: 大文件分片上传及断点续传
作为 **普通用户**，我希望能够 **向便签上传体积较大的视频或文件，且如果网断了能恢复上传**，以便于 **管理大型设计稿或录屏说明。**
- **验收标准 (AC):**
  - [ ] 支持拖拽上传和点击按钮上传文件。
  - [ ] 超过 10MB 的文件前端自动按需切片（如 2MB/片）上传。
  - [ ] 如果中断或刷新页面，再次上传相同文件时，能够执行“秒传”或“续传”。
  - [ ] 必须显示当前进度条和成功/失败状态。

### Story 3.2: 附件在线预览与播放
作为 **普通用户**，我希望能够 **直接在便签中点击预览图片或播放音视频，而不需要每次都下载**，以便于 **快速核阅资料。**
- **验收标准 (AC):**
  - [ ] 对于 `.jpg`, `.png` 等图片类型，附件列表项展现略缩图，点击后全屏画廊预览（Lightbox）。
  - [ ] 对于 `.mp4`, `.mp3` 等类型，点击后弹窗展示系统原生或自定义播放器流式播放。
  - [ ] `.doc`, `.xls` 等文稿，根据配置可选择浏览器新页签打开或提示仅限下载。

## Epic 4: 视图总览与日历模式
**描述：** 用户需要跳出单一便签，从宏观维度或时间维度审查自己的任务密度。

### Story 4.1: 日历月/周视图映射
作为 **普通用户**，我希望能够 **切换至“日历视图”，直观地看到每个日期格子里有哪些截止的任务**，以便于 **统筹本周/本月的个人精力分配。**
- **验收标准 (AC):**
  - [ ] 提供月视图（网格）和周视图（时间轴或分列）切换。
  - [ ] 便签以对应颜色的“条块/胶囊”形式渲染在它们各自的“截止日期”那一天。
  - [ ] 在日历视图中点击条块，能够弹出的浮层中实现快览便签摘要和快速修改状态。

### Story 4.2: 便签卡片的拖拽排期
作为 **普通用户**，我希望能够 **在日历视图中，直接将某天的便签拖拽到另一天**，以便于 **任务由于不可抗力延期时能瞬间完成改期。**
- **验收标准 (AC):**
  - [ ] 支持拖拽事件，放下后触发后端接口更新该便签的`截止日期`及记录操作事件。

## Epic 5: 用户协作与权限基础
**描述：** V1.2 的协作规划。

### Story 5.1: 基础认证与登录拦截
作为 **普通用户**，我希望 **我的便签是私密和受账号保护的，需要注册并登录才能访问**，以便于 **工作隐私不被陌生人访问。**
- **验收标准 (AC):**
  - [ ] 提供账户密码注册/登录页面。
  - [ ] 登录状态具有时效性（如 JWT 7 天），过期强制踢回登录页。
  - [ ] 后端严格校验数据越权：用户 A 无法通过拼接 URL 请求或查看用户 B 的非共享便签接口和附件。

## Epic 6: 视图交互优化
**描述：** 保证主工作区的信息密度适中，用户可按需加载，避免一次性渲染过多内容。

### Story 6.1: 主工作区渐进式折叠加载
作为 **普通用户**，我希望 **打开应用时默认只看到前10条最重要的便签，其余通过点击或滑到底部再展示**，以便于 **首屏信息不过载，专注于最紧要的任务。**
- **验收标准 (AC):**
  - [ ] 进入主工作区（及各分区），默认仅渲染且展示前 **10 条**便签。
  - [ ] 第10条下方显示"**展开更多（还有 N 条）**"按钮，N 动态计算剩余数量。
  - [ ] 点击"展开更多"后，追加展示下一批 **10 条**，不刷新整页。
  - [ ] 将页面滚动到最底部时，自动触发加载下一批 10 条（无限滚动），无需手动点击按钮。
  - [ ] 所有条目加载完毕后，按钮消失，底部显示"**已加载全部 N 条**"提示。

### Story 6.2: 历史记录多维度统计分析
作为 **普通用户**，我希望 **在历史记录分区查看我的任务完成趋势、处理效率和热力图**，以便于 **定期复盘，了解自己的生产力模式。**
- **验收标准 (AC):**
  - [ ] 进入"历史记录"分区，顶部有"📊 展开统计分析"按钮，默认收起以节省屏幕空间。
  - [ ] 展开后，展示6类图表：完成趋势（折线/柱状）、按颜色分类饼图、月度热力图、平均处理时长条形图、按标签排行榜、逾期完成率数字卡片。
  - [ ] 提供时间筛选器：本周 / 本月 / 本年 / 自定义区间，切换后图表与下方便签列表**联动刷新**。
  - [ ] 已完成便签卡片上展示"完成时间"和"处理耗时（天）"标签。
