# 智能便签任务管理系统 —— UI/UX 设计指引与低保真原型说明

> **文档说明**
> 本文档定义了智能便签系统 Web 端的核心视觉规范，并通过文字结构树描绘了关键页面的交互骨架。供前端开发及界面设计师进行高保真 UI 设计时作为准则。

---

## 1. 整体视觉风格与色彩规范 (UI)

产品定位为“高效、聚焦、轻量”的效能工具。设计语言应贴近现代 SaaS 软件（如 Trello, Notion, Todoist）。

*   **主色调 (Primary Color):** `蓝/靛青`（代表专业和冷静）。
*   **背景色 (Background):** `浅灰白 (#F4F5F7)`，用于区分纯白的便签卡片，产生悬浮感。
*   **卡片投影 (Shadow):** 采用大扩散半径、低透明度的柔和阴影，避免生硬的边框。
*   **圆角 (Border Radius):** 全全局使用中等圆角（如 `8px` 或 `12px`），传递亲和力。
*   **字体 (Typography):** 优先使用系统无衬线字体（如 `Inter`, `Roboto`, `PingFang SC`），保证易读性。层级分明，便签标题加粗，正文常规。

### 1.2 便签颜色语义系统 (对应需求中的固定颜色)
用户在创建便签时选择颜色，系统赋予以下语义，UI 渲染时应采用**浅色背景 + 深色文字**或**左侧粗条色带**的设计，避免高饱和度大面积色块刺眼。
*   🔴 **紧急/高优:** `鲜红/浅粉背` (如：Blocker, 今日必做)
*   🟠 **重要/预警:** `亮橙/浅橙背` (如：核心里程碑)
*   🔵 **常规/日常:** `静蓝/浅蓝背` (默认色)
*   🟢 **低优/提示:** `翠绿/浅绿背` (如：备忘, 学习)

## 2. 核心页面结构与交互原型 (UX)

### 2.1 全局框架 (Layout)
采用经典的 `左侧边栏 (导航) + 顶部栏 (全局控制) + 右侧主内容区` 结构。

```text
[浏览器窗口]
├── 左侧边栏 (固定宽度, 可折叠)
│   ├── 用户头像与昵称
│   ├── 快速新建按钮（醒目大按钮）
│   ├── ── 主导航 ──
│   ├── 📋 主工作区（进行中 + 草稿）      ← 默认高亮入口
│   ├── ── 次级导航 ──
│   ├── 🕐 历史记录（已完成 / 已归档）
│   ├── ⏸  悬挂列表 [N]                   ← 角标显示挂起数量
│   ├── 📅 日历视图
│   ├── 📊 统计图表
│   └── ── 底部 ──
│       ├── 🗑️  垃圾桶（已删除，保留30天）
│       └── ⚙️  系统设置 / 退出登录
│
├── 顶部控制栏（吸顶）
│   ├── 当前分区标题（如"主工作区"/"历史记录"）
│   ├── 搜索框（全局全文检索，覆盖所有分区）
│   └── 快捷筛选器（按颜色、优先级筛选）
│
└── 核心内容区（随导航切换不同分区内容）
    └── [ 视图：主工作区 / 历史记录 / 悬挂列表 / 日历 / 统计 / 垃圾桶 ]
```

### 2.2 主工作区：便签列表（进行中 / 草稿）
核心交互：展示所有**活跃中**的便签，按优先级 → 创建时间排序，已完成/已删除/挂起的便签**不在此区域**。

```text
[核心内容区 - 瀑布流布局]  ←  当前分区：主工作区
│
├── [便签卡片 A] (红色主题) ← 优先级：紧急，排第一
│   ├── (左上) 🔴 紧急          (右上) 截止：03-05（若有）
│   ├── 标题：准备 Q1 季度汇报 PPT（加粗）
│   ├── 正文前2行摘要...
│   ├── [子任务进度条] 1/3
│   └── 底部快捷操作：[✅ 完成] [⏸ 挂起] [🗑 删除]
│
├── [便签卡片 B] (橙色主题) ← 优先级：重要
│   ├── 标题：购买服务器资源
│   └── 无截止日期（不在日历显示）
│
└── [便签卡片 C] (蓝色主题 - 草稿态)
    ├── 标题：[草稿] 周报整理思路       ← 角标或浅色标识草稿状态
    └── 底部快捷操作：[✅ 完成] [⏸ 挂起] [🗑 删除]

─── 展示 3/10 条（已到达默认加载上限）───────────────
│
└── [  ▼ 展开更多（还有 N 条）  ]     ← 点击加载下一批10条
    或向下滚动到此处自动加载 →
    加载完毕后显示：已加载全部 N 条 ✓
```

> **注意**：已完成便签点击"✅ 完成"后**立即从本视图消失**，流转至"历史记录"分区，主工作区始终保持清爽。

### 2.2b 历史记录分区（点击左侧「历史」图标后展示）

```text
[核心内容区 - 当前分区：历史记录]
│
├── ── 统计分析看板（默认折叠）──────────────────────────
│   [📊 展开统计分析 ▼]
│   展开后显示：
│   ├── [完成趋势折线图]  [按颜色分类饼图]
│   ├── [月度热力图]      [平均处理时长条形图]
│   ├── [按标签排行榜]    [逾期完成率: 12%]
│   └── 时间筛选：[本周] [本月] [本年] [自定义]
├── ──────────────────────────────────────────────────
│
├── [便签卡片 A] (灰色-已完成)
│   ├── ~~准备 Q1 季度汇报 PPT~~  (删除线)
│   ├── 完成时间：2026-03-01  处理耗时：3天
│   └── 操作：[↩ 重新激活]
│
├── [便签卡片 B] (已归档)
│   └── ~~购买服务器资源~~  归档时间：2026-02-20
│
└── [  ▼ 展开更多（还有 N 条）  ]    ← 同主工作区分页加载逻辑
```

### 2.3 便签详情侧边抽屉 (Detail Drawer)
当点击某张便签卡片或“新建”按钮时，**从屏幕右侧滑出抽屉（Drawer）**，而非跳转新页面，以保持用户对列表的上下文感知。

```text
[右侧滑出抽屉 - 占屏幕宽 1/3 到 1/2]
├── (顶部操作栏) [保存]  [状态下拉菜单 ▼]  [关闭 ✕]
│
├── (表单区 - 上半部分)
│   ├── [颜色选择器] (红/橙/蓝/绿 圆点)
│   ├── [标题输入框] (大字体, Placeholder: "准备做什么？")
│   ├── [日期选择器] (必填, 默认当天)
│   └── [Markdown 正文编辑器] (支持富文本展示)
│
├── (子任务区 - 中部)
│   ├── "子任务 (进度: 1/3)"
│   ├── [✓] 审核前端代码 (缩进1层)
│   ├── [ ] 审核后端代码 (缩进1层)
│   │   └── [ ] 检查 SQL 注入 (缩进2层)
│   │   └── [ ] 检查 JWT 逻辑 (缩进2层)
│   └── [+ 添加子任务] (文字按钮)
│
├── (附件区 - 下半部分)
│   ├── "附件区域"
│   ├── [拖拽上传虚线框 / 点击上传按钮]
│   ├── [图片.png] (带略缩图，Hover显示预览图标)
│   └── [演示.mp4] (带播放图标)
│
└── (活动时间线区 - 底部折叠面板)
    └── 展开查看：[09:00 UserA 创建] -> [10:30 增加附件] ...
```

### 2.4 日历视图 (Calendar View)

```text
├── 顶部：[ < ]  2026年 3月  [ > ]  [本月] [本周]
│
├── 周一           周二          周三          周四  ...

```

## 3. 关键交互约束说明

1.  **防止数据丢失：** 用户在“便签详情抽屉”中输入内容时如果误点外部遮罩层关闭，系统必须弹窗拦截：“有未保存的更改，确认离开吗？”。
2.  **断点续传感知：** 大文件上传时，如果用户关闭抽屉，只要不关闭浏览器刷新，左下角应有一个悬浮的“上传列队进度条”继续工作。
3.  **响应式降级：** 屏幕宽度 `< 768px` 时，左侧边栏默认隐藏（汉堡菜单呼出），瀑布流降级为单列列表，右侧滑出抽屉改为底部向上弹出的全屏浮层。
