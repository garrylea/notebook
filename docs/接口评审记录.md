## 1. 认证模块 (Auth)

### 页面关联
- 登录页：`POST /auth/login`
- 注册页：`POST /auth/register`
- 全局 Token 无感刷新（axios 拦截器）：`POST /auth/refresh`
- 顶部栏退出登录按钮：`POST /auth/logout`

### 字段核对

| 接口       | 原型页面绑定字段                   | API 文档字段                                                 | 核对结论 |
| ---------- | ---------------------------------- | ------------------------------------------------------------ | -------- |
| 注册       | 邮箱输入框、密码输入框、昵称输入框 | `email`, `password`, `username`                              | ✅ 一致   |
| 登录       | 邮箱输入框、密码输入框             | `email`, `password`                                          | ✅ 一致   |
| 登录响应   | 存储 Token → 页面跳转主列表        | 响应体含 `accessToken`、用户信息；`refreshToken` 写入 Cookie | ✅ 满足   |
| 刷新 Token | 无 UI 触发（拦截器自动）           | Cookie 自动携带，无需额外参数                                | ✅ 满足   |
| 登出       | 退出登录按钮                       | 无请求体                                                     | ✅ 满足   |

### Action Items

> **AI-1** 【✅ 已处理】：已在接口设计文档"认证接口响应体示例"章节中补充 `POST /auth/login`、`POST /auth/register`、`POST /auth/refresh` 的完整响应 JSON，明确 `avatar_url`、`preferences`、`status` 等字段结构。

## 2. 便签模块 (Notes)

### 页面关联
- **主工作区**（List View）→ `GET /notes`
- **便签卡片底部操作（标记完成、删除）** → `PATCH /notes/:id/status`、`DELETE /notes/:id`
- **右侧详情抽屉（新建/编辑）** → `POST /notes`、`PUT /notes/:id`
- **抽屉顶部状态下拉菜单** → `PATCH /notes/:id/status`
- **回收站恢复/彻底删除** → `POST /notes/:id/restore`、`DELETE /notes/:id/force`

### 字段核对

| 接口             | 原型绑定字段                                                      | API 文档字段                                                                  | 核对结论 |
| ---------------- | ----------------------------------------------------------------- | ----------------------------------------------------------------------------- | -------- |
| **获取便签列表** | 卡片展示：标题、状态标签、截止日期、颜色色带、子任务进度条（x/y） | 已补充完整 List Item JSON                                                     | ✅ 已补充 |
| **创建便签**     | 颜色选择器、标题框、日期选择器（选填）、正文                      | `title`(必填), `color`(必填), `due_date`(选填), `content`, `tags`, `priority` | ✅ 完整   |
| **更新便签**     | 同创建                                                            | 允许更新 `title`, `content`, `color`, `priority`, `tags`                      | ✅ 满足   |
| **更新便签状态** | 状态下拉菜单 + 卡片快捷"标记完成"按钮                             | `status`, `remark`                                                            | ✅ 满足   |
| **移入回收站**   | 卡片底部删除图标                                                  | 软删除，设 `deleted_at`                                                       | ✅ 满足   |

### Action Items

> **AN-1** 【✅ 已处理】：已在接口设计文档中补充 `GET /notes` 完整 List Item 响应 JSON，包含 `subtask_total`、`subtask_completed`、`has_attachments` 等所有卡片渲染所需字段。

> **AN-2** 【✅ 已处理】：已在 `GET /notes/:id` 详情响应体中内嵌 `activity_logs` 数组，包含 `action_type`、`operator`、`detail`（前后状态差异）、`created_at` 字段，直接支持时间线 UI 渲染。

> **AN-3** 【✅ 已处理】：已在接口设计文档"便签接口补充说明"中列出 `sort_by` 全部枚举值（`priority_desc`、`created_at_desc`、`due_date_asc` 等7种），默认组合为 `priority_desc,created_at_desc`。

## 3. 子任务模块 (Subtasks)

### 页面关联

- 勾选子任务完成 → `PUT /notes/:noteId/subtasks/:subtaskId`
- 拖拽排序 → `PATCH /notes/:noteId/subtasks/reorder`

### 字段核对

| 接口           | 原型绑定字段                           | API 文档字段                                            | 核对结论 |
| -------------- | -------------------------------------- | ------------------------------------------------------- | -------- |
| 添加子任务     | 文字输入框 + 可选子级层级（缩进1-2层） | `title`, `status`, `assignee`, `due_date`, `parent_id`  | ✅ 满足   |
| 更新子任务     | 勾选框状态切换                         | `title`, `status`, `assignee`, `due_date`, `sort_order` | ✅ 满足   |
| 删除子任务     | Hover 删除按钮                         | 路径参数                                                | ✅ 满足   |
| 子任务批量排序 | 拖拽后自动触发                         | `[{ id, sort_order }]` 数组                             | ✅ 满足   |

### Action Items

> **AS-1** 【✅ 已处理】：**决策确认**：子任务数据直接内嵌在 `GET /notes/:id` 响应体的 `subtasks` 字段中，采用树形嵌套结构（`children` 数组），最多 3 层。不提供独立分页接口，已在接口设计文档详情响应体示例中明确标注。

## 4. 附件模块 (Attachments)

### 页面关联
- **抽屉下半"附件区域"**：拖拽上传 / 点击上传
- **悬浮上传队列进度条**（关闭抽屉后仍继续上传）
- 图片缩略图预览、视频播放按钮

### 字段核对

| 接口           | 原型绑定场景                      | API 文档字段                                    | 核对结论 |
| -------------- | --------------------------------- | ----------------------------------------------- | -------- |
| 小文件直传     | < 5MB 文件                        | `multipart/form-data`, `files`                  | ✅ 满足   |
| 初始化分块上传 | ≥ 5MB 文件 → Web Worker 计算 Hash | `filename`, `file_size`, `total_chunks`, `hash` | ✅ 满足   |
| 上传文件分块   | 循环上传剩余 chunk                | `chunk_index`, `chunk_data`                     | ✅ 满足   |
| 合并文件分块   | 所有 chunk 传完后调用 merge       | `filename`, `total_chunks`, `hash`              | ✅ 满足   |
| 删除附件       | Hover 出现删除按钮                | 路径参数                                        | ✅ 满足   |
| 附件静态鉴权   | Nginx auth_request 回调           | Cookie/Header 转发                              | ✅ 满足   |

### Action Items

> **AA-1** 【✅ 已处理】：已在接口设计文档"附件模块补充说明"中补充小文件直传和分块合并的完整响应 JSON，包含 `id`、`filename`、`file_type`、`file_size`、`url`、`md5_hash`、`created_at` 全部字段。

> **AA-2** 【✅ 已处理】：已统一标注上传阈值为 **5 MiB**（`5 * 1024 * 1024` 字节），分块大小为 **2 MB**，以表格形式写入接口文档附件模块说明中，前后端共同遵守。
